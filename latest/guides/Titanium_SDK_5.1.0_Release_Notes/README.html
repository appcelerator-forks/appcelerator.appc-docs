Notice of Feature and Behavior Changes

Android Platform

iOS Platform

Ti.Database.ResultSet fieldCount function has been turned into a property.

Windows Platform

New Features and Improvements

Android Platform

Fixed memory leak from imageAsResize function.

LiveView doesn't throw errors if action bar is implemented in the app.

HTTPClient allows contentType that is unspecified in the response.

iOS Platform

Added documentation for the Ti.UI.TableViewRow properties header and footer.

Resizing a view now keeps child labels centered.

On iOS6, insertRowBefore no longer animates when inserted.

Label.accessibility label works properly inside a ScrollView.

On iOS7, when using AccessibilityLabel as a leftNavButton or rightNavButton now appears as a button.

System Info icon is now consistent in size between toolbar and toolbar and toolbar and a view.

LiveView now longer throws errors while evaluating Android conditional code on iOS.

Remote ImageViews now load while scrolling through ListView.

Connecting Ti.Network to pubnub doesn't crash on iOS9 devices.

The default height of the UITableView section header is now 29px.

Removed "pcm missing" warnings.

Making changes to a newly build app now shows updates in iOS simulations.

Setting setshowUndoRedoActions to true after creation no longer crashes app.

Suppressed missing UIBackgroundModes for "fetch" and "remote-notification" warnings.

Windows Platform

Added support for Visual Studio 2015, compile targeting Windows 10 Universal, and DefaultIcon.png.

Allow mixing Ti.Ui.View and Windows UI components.

Implemented Ti.Filesystem.FileStream, Ti.Stream, Ti.IOStream, BlobStream and BufferStream, Ti.Buffer, Ti.Codec, and Ti.UI.Picker.

Implemented Ti.App._restart() for LiveView.

Support for array of views on Ti.UI.View.add.

Improved font lookup and cache.

Update JavaScript runtime to be more stability.

Ti.Database.ResultSet fieldName() function no longer crashes.

Ti app build error isn't ignored from CLI.

Fixed incompatible behavior on touch events.

Ti.UI.ListItem height and color can now be set.

MSBuild processes are no longer orphaned after building a project.

CLI now exits when it encounters a parse error.

Known Issues

Android

Camera and Filesystem Permissions

The app will do nothing if:

  • If filesystem permission is granted and camera permission not granted

  • If camera permission is granted and filesystem permission not granted

If both filesystem and camera are granted permission, the app performs as normal.

HTTPClient PATCH Method for 5.x.x

We don't officially support PATCH method but we do offer a workaround of OkHTTPClient.
